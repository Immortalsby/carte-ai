# 产品简报：CarteAI

## 执行摘要

每年有 1 亿游客到访法国。他们坐进一家餐馆，打开一本看不懂的菜单，不知道点什么。服务员不一定会讲他们的语言，更不可能花 5 分钟给每桌解释每道菜。结果？游客靠运气点单，错过好菜，餐馆错过了推高利润菜和提升满意度的机会。

**CarteAI 是一张贴在桌上的 QR 码背后的 AI 点餐顾问。** 顾客扫码 → 选"第一次来 / 10 欧以内 / 招牌菜 / 健康 / 分享" → 3 秒钟出 3 道个性化推荐，附带价格、过敏原警告和理由——全程 19 种语言，无需注册，无需打字。

更聪明的是：CarteAI **根据顾客与餐馆的文化匹配自动切换推荐策略**。外国游客在中餐馆？AI 用他的语言翻译菜名、推荐入门菜。中国人在中餐馆？AI 切换成组菜模式——5 个人吃饭推荐几个热菜几个凉菜，问有没有人不吃香菜。同一个产品，两种体验。

餐馆老板不需要学新系统。上传一张菜单照片，AI 自动提取结构化数据。菜单有变？给 WhatsApp 机器人发条消息："今天牛排没了"——搞定。

CarteAI 不是电子菜单，不是聊天机器人，也不是支付工具。它是欧洲唯一一个**以 AI 推荐为核心**的餐桌级产品，在西欧没有直接竞品。

## 问题

**顾客侧**：面对外语或复杂菜单（中餐 80 道菜、法餐的专业术语、印度菜的香料组合），第一次来的顾客几乎无法做出好的选择：

- 游客不懂当地语言，菜名翻译了也不知道什么味道
- 有过敏原 / 饮食限制但不知道怎么用外语表达
- 有预算但不知道哪些菜值得点
- 服务员在高峰时段没时间逐桌解释
- 本地人去异国菜餐厅同样困惑（法国人进中餐馆也不知道点什么）

**老板侧**：菜单数字化门槛高（拍照→结构化→多语言→过敏原标注→持续更新），现有方案（Menutech €15-180/月）只做过敏原标注和翻译，不做推荐，不帮老板多赚钱。老板真正想要的不是"数字菜单"，而是**每桌多赚 €2-5**。此外，EU 1169/2011 法规要求所有餐饮企业必须以书面形式向消费者展示 14 种过敏原信息——数字菜单必须满足与纸质菜单相同的标注要求，不合规面临罚款风险。

**现状**：顾客靠运气或 TripAdvisor 评论点单。老板用纸质菜单或静态 PDF 二维码。推荐全靠服务员口头推销——不可规模化、不可追踪、不可多语言。

## 方案

### 顾客体验（3 步 / 10 秒）

1. **扫码** → 进入 `carte-ai.link/r/餐馆slug`
2. **点预设按钮**（"第一次来""≤10€""不辣""分享"）+ 勾选限制（素食/无麸质/不吃牛肉）
3. **看 AI 推荐** → 3~4 道菜，含本地化理由、价格、过敏原提醒、信心评分

无需注册。自动检测浏览器语言。支持语音输入。

**智能模式切换**：当顾客的浏览器语言与餐馆菜系语言匹配时（如中国人进中餐馆），自动切换为"组菜顾问"模式——不再侧重翻译和入门推荐，而是根据人数、口味偏好（不吃香菜、不吃辣）推荐热菜凉菜搭配，像一个懂行的朋友帮你点菜。

### 老板体验（零摩擦录入）

- **初始录入**：拍菜单照片 / 上传 PDF → AI 自动提取 → 老板审核确认（不是从零录入，是审批一份草稿）
- **日常更新**：WhatsApp 对话式管理——"删掉牛排""加这个新菜 [发图]"——**像跟真人助理对话一样**。这是 CarteAI 的杀手锏功能：法国小老板极度抗拒登录后台系统，但他们每天都在用 WhatsApp 管理订货和排班。
- **数据反馈**：扫码率、推荐被查看次数、顾客语言分布

### 推荐采纳追踪

顾客浏览推荐页一段时间后，页面温和弹出："您刚才点了我们推荐的菜吗？"选"是/否"——轻量级、不强制、但能采集到核心转化数据。结合扫码量和停留时长，构成完整的 ROI 证明链。

### 技术核心

- **推荐引擎**：本地规则（必跑/免费）+ LLM 增强（GPT-4.1-nano，每次扫码 $0.00024），双层架构保证永远有结果
- **LLM 安全护栏**：6 条硬约束——不发明菜品、不编造价格/过敏原/卡路里，过敏数据缺失时强制提醒找服务员
- **14 种 EU 过敏原类型内置**——直接满足 EU 1169/2011 数字标注义务，为餐馆提供合规保护。每次过敏原查询自动存档，可作为合规审计证据。
- **极低 LLM 成本**：nano 模型即使繁忙大店（9000 扫/月）也只需 $2.17/月，毛利 >85%，留出巨大的定价弹性和免费试用空间
- **数据隐私**：仅采集匿名行为数据（扫码/语言/推荐点击），不收集个人身份信息，符合 GDPR 匿名数据处理原则

**护城河**：

1. **19 语言 + 文化感知推荐引擎** —— 不是翻译，是根据顾客文化背景切换整套推荐策略
2. **marginPriority 餐馆经济学** —— 帮老板推高利润菜 = 直接帮他多赚钱 = 最硬的付费理由
3. **WhatsApp 原生菜单管理** —— 零学习成本，是法国独立餐馆老板最自然的工作界面
4. **EU 过敏原合规盾牌** —— "不用 CarteAI，你的数字菜单可能不合规"

## 用户画像

### 买单者：独立餐馆老板（巴黎起步）

- 菜单复杂或有语言障碍（中餐/印度菜/法餐专业术语）
- 旅游客流高的区域（1-4区、拉丁区、蒙马特……）
- 想数字化但没时间学新系统
- 核心动机：**每桌多赚钱** + 给外国顾客留好印象

### 终端用户：第一次来的顾客

- **外国游客**：不懂当地语言，需要翻译 + 入门推荐
- **本地异国菜探索者**：法国人进中餐馆也不知道点什么
- **同文化聚餐**：中国人在中餐馆——不需要翻译，需要组菜搭配建议
- 共同点：有饮食限制（过敏原/素食/预算），想快速做出好选择

### Aha Moment

- **外国顾客**：扫码 → 点"第一次来" → 3 秒出母语推荐 → "比问服务员还快还准"
- **同文化顾客**：扫码 → 识别到中文 → 直接问"几个人？有忌口吗？" → 推荐热菜凉菜搭配 → "这才是点菜该有的样子"
- **老板**：看到第一周数据 + 收到顾客软反馈 → "这东西真的有用"

## 范围

### Phase 1（POC + 商业化，0-6 个月）

**IN**：

- `/r/[slug]` 餐馆专属页面
- 菜单持久化（Cloudflare KV / Vercel Postgres）
- 基础鉴权（老板端）
- 速率限制 + LLM 成本封顶（按餐馆绑定上限）
- 默认 GPT-4.1-nano（推荐）+ Haiku 4.5（OCR 提取）
- 文化感知模式切换（外客模式 vs 同文化组菜模式）
- 海报生成（真实 QR）+ "AI Menu · 19 Languages" 门口标识
- `marginPriority` 老板开关
- `contact@carte-ai.link` 咨询入口（Cloudflare Email Routing）
- 基础埋点（扫码/推荐/语言/停留/采纳弹窗）
- 餐后柔性好评引导（非侵入式，推荐页底部小提示）
- 核心 E2E 测试（Playwright）

**OUT**：

- 自助注册 / onboarding 向导
- Stripe 计费集成
- 完整 analytics dashboard
- OAuth / SSO
- POS 集成
- 多语言海报模板定制

### Phase 1.5（WhatsApp Agent，6-9 个月）

- Hermes Agent + WhatsApp Business API Gateway
- 对话式菜单 CRUD（文字 + 图片识别）
- Pro 阶梯定价解锁

### Phase 2（开放 SaaS，9-12 个月）

- 自助注册 + onboarding
- Stripe 订阅
- 多店管理
- 完整 BI / 数据看板

## 商业模型

| 档位 | 月费 | 包含 | LLM 成本 | 毛利 |
|------|------|------|---------|------|
| **Starter** | €19/月 | QR 推荐 + 静态菜单 + 海报 + 埋点 + 过敏原合规 | ~$0.5-3 | >85% |
| **Pro** | €39/月 | + WhatsApp Agent 实时更新 + 日报 | ~$3-8 | >80% |
| **Enterprise** | 定制 | + API / POS 集成 + 多店 + 专属支持 | 按量 | — |

**获客策略**：POC 期间全免费 → 商业化后首月免费试用（LLM 边际成本趋近零，完全可承受）→ 口碑传播 + 小红书/社交媒体内容营销。

**成本封顶**：按餐馆绑定 LLM 费用上限。Starter 超 5,000 扫/月自动降级纯本地规则（体验不中断，只是少了 LLM 增强文案）；Pro 有软上限 + 通知。

**增长引擎**：初期靠口口相传 + 创始人社交媒体（小红书等）+ "AI Menu" 门口标识的线下曝光。每家已入驻的餐馆门口标识本身就是广告——隔壁老板看到会问"这是什么"。

## 愿景

如果 CarteAI 做成了：**欧洲每一张餐桌上的 QR 码背后都是 CarteAI**。

第一步是推荐。第二步是 WhatsApp 菜单管理。第三步是从推荐延伸到点单 → 支付 → 会员 → 复购——做餐馆的 AI 前台。Sunday 做了支付层，Zenchef 做了运营层，CarteAI 做推荐层——最终这三层会合并，而**推荐是离顾客决策最近的那一层**。

## 风险与缓解

| 风险 | 严重 | 缓解 |
|------|------|------|
| QR 扫码率低（海报不吸引人） | 🔴 | POC 快速迭代海报设计；A/B 测试不同话术 |
| 老板觉得"没用" | 🔴 | 首月免费 + 每周面对面 review 数据 |
| LLM 成本失控（大店） | 🟡 | 按餐馆绑定上限 + nano 模型 + 本地规则兜底 |
| WhatsApp Business API 审批复杂 | 🟡 | Phase 1 不依赖（先用 admin 后台）；1.5 再接入 |
| 竞品跟进（Sunday 加推荐功能） | 🟡 | 文化感知模式切换 + 19 语言深度 = 不是一两周能抄的 |
| 单城市风险（巴黎依赖旅游季） | 🟢 | POC 验证后快速扩到里昂/马赛/其他旅游城市 |
